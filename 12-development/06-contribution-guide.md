# コントリビューションガイド

## コントリビューションの流れ

1. **Issue を確認/作成**
2. **ブランチを作成**
3. **コードを書く**
4. **テストを書く**
5. **PR を作成**
6. **レビューを受ける**
7. **マージ**

## Pull Request の作成

### PRテンプレート

```markdown
## 変更の概要
このPRで何を変更したか簡潔に説明

## 変更の目的
なぜこの変更が必要か

Closes #123

## 変更内容の詳細
- 変更点1
- 変更点2

## テスト方法
1. ステップ1
2. ステップ2

## チェックリスト
- [ ] テストが全て通過
- [ ] Lintエラーなし
- [ ] ドキュメント更新済み
```

### PRサイズ

- **理想**: 300行以下
- **最大**: 500行以下
- 大きい場合は複数のPRに分割

## コードレビュー

### レビュワーへの依頼

- 最低1名のレビュワーをアサイン
- 重要な変更は2名以上推奨

### レビューコメントへの対応

- **必須修正**: 必ず対応
- **推奨修正**: 合理的な理由があれば説明して次回対応も可
- **質問**: 丁寧に回答

## コーディングチェックリスト

### 実装前

- [ ] Issue を作成/確認
- [ ] 設計を検討
- [ ] 影響範囲を確認

### 実装中

- [ ] コーディング規約に従う
- [ ] 型ヒント/型定義を追加
- [ ] エラーハンドリングを実装
- [ ] ログ出力を追加

### 実装後

- [ ] テストを追加/更新
- [ ] ドキュメントを更新
- [ ] セルフレビュー
- [ ] Lint/Format実行

## テストの追加

```python
# 新機能にはテストを追加
def test_new_feature():
    result = new_feature("input")
    assert result == "expected"
```

## ドキュメントの更新

- README.md
- API ドキュメント
- 関連ドキュメント

## 質問・サポート

- **GitHub Discussions**: 質問・議論
- **GitHub Issues**: バグ報告・機能要望
- **Slack**: リアルタイムコミュニケーション
