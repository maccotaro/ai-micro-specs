# 00. システム全体概要

このディレクトリには、AI Micro Serviceシステム全体のアーキテクチャと設計に関するドキュメントが含まれています。

## 📄 ドキュメント一覧

### 01. システム全体アーキテクチャ設計書
`01-system-architecture.md`

システム全体の構成、7つのマイクロサービスの役割、サービス間の関係性を説明します。

**主な内容**:
- システム概要
- マイクロサービス構成
- アーキテクチャパターン（BFF、マイクロサービス）
- 技術選定理由

### 02. マイクロサービス連携設計書
`02-microservices-integration.md`

各サービスがどのように連携し、データをやり取りするかを定義します。

**主な内容**:
- サービス間通信プロトコル
- データフロー
- 依存関係
- サービスディスカバリ

### 03. インフラ構成設計書
`03-infrastructure.md`

システムのインフラストラクチャ、ネットワーク構成、デプロイ環境を説明します。

**主な内容**:
- Docker構成
- ネットワークトポロジ
- ポートマッピング
- 環境構成（開発・本番）

### 04. 技術スタック一覧
`04-technology-stack.md`

システムで使用している技術スタックの一覧と選定理由を記載します。

**主な内容**:
- フロントエンド技術（Next.js、React、TypeScript）
- バックエンド技術（FastAPI、Python）
- データベース（PostgreSQL、Redis）
- その他のライブラリとツール

## 🎯 このセクションを読むべき人

- **全ての開発者**: システム全体像を把握するために最初に読んでください
- **アーキテクト**: システム設計の意思決定を理解するため
- **新規参加者**: プロジェクトのオンボーディングとして
- **インフラ担当者**: システムの構成と依存関係を理解するため
